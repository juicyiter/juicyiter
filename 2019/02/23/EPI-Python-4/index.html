Be aware that commutativity and associativity can be used to perform operations in parallel and reorder operations.

Compute The Parity of A Word

The parity of a binary word is 111 is the number of 111s in the word is odd; otherwise, it is 000. For example the parity of 101111011110111 is 000, and the parity of 101110111011 is 111.

How would you calculate the parity of a very large number of 646464-bit words?

The parity of a word is irrelevant with the order of the bits. So, suppose you’ve pre-computed a parity table of, let’s say, 161616 bits. (Since 65536​65536​65536​ is a relative small number. )

Reverse Bits

Write a program that takes a 646464-bit unsigned integer and retums the 646464-bit unsigned integer consisting of the bits of the input in reverse order. For example, if the input is (1110000000000001)(1110000000000001)(1110000000000001), the output should be (1000000000000111)(1000000000000111)(1000000000000111).

To implement reverse when the operation is to be performed repeatedly, we look more carefully at the structure of the input, with an eye towards using a cache.

For example, to reverse a 888-bit integer, we use the 222-bit lookup table keys. The table is rev = <(00), (01), (10), (11)>. If the input is (10010011)​(10010011)​(10010011)​, its reverse is rev(11), rev(00), rev(01) and rev(10).

The time complexity is same as Lookup table

Find A Closest Integer With The Same Weight

You can assume xxx is not 000, or all 111s. For example, if x=6x = 6x=6, you should return 555. You can assume the integer fits in 646464 bits.

Suppose we flip the bit at index k1k_1k1​ and flip the bit at index k2k_2k2​, k1>k2k_1 > k_2k1​>k2​. Then the abosolute value of the difference of the original integer and the new one is 2k1−2k22^{k_1} - 2^{k_2}2k1​−2k2​. To minimise it, we should make k1k_1k1​ as small as possible and k2k_2k2​ as close to k1k_1k1​.

The bit at index k1k_1k1​ must be different from the bit in k2k_2k2​, this means the smallest k1k_1k1​ is the rightmost bit that differs from the LSB, and k2k_2k2​ must be the very next bit.

In summary, the correct approach is to swap the two rightmost consecutive bits that differ.
