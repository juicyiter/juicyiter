Hexo 入坑笔记

发表于 05-05-2018 | 更新于: 07-25-2018 | 分类于 Note
| 字数统计: | 阅读时长 ≈

之前在 GitHub 上搭建过一个个人网站，那次是根据 HTML5 UP 上的包自己修改完，然后传到 GitHub 上的，觉得很简陋，所以打算入坑 Hexo ，嘻嘻。

写在前面

网上这么多 Hexo 建站的教程了，我这篇也不算教程，毕竟自己还太菜，对前端也只能删和改。这个就算个笔记吧，毕竟好记性不如烂笔头，希望读者能避免我踩过的坑，在建站的路上走得更平稳，更快速，嘿嘿。

因为我之前的静态博客是托管在 GitHub 上的，不过无奈每次打开都有点慢，不知道是 DNS 的缘故，还是访问 GitHub 太慢，对 DNS 了解不深的我比较倾向后者，而且「码云」的免费容量为 5G，虽然不能绑定域名，但是国内访问速度真是飞起 ，所以我这次把博客托管在「码云」。（ 将其托管在 GitHub 步骤一样）

Hexo 是啥

打开 Hexo 官网，赫然写着下面一句话

A fast, simple & powerful blog framework

我觉得很精辟了。我之前增加新的页面的时候，要把模版从头至尾改一遍，然后在根目录的 index.html 增加链接新页面的 href ，真是神烦了。当时建了有8、9个页面的样子，然后加上毕业一堆糟心事儿，就搁置了一个多月没更新。现在好了，用 Hexo 几句代码就能更新，上传，而且有了主题之后增加新页面也很简单，具体见下篇笔记~。
