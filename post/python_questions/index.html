Garbage Collection

Garbage collection is the process of finding data objects in a running programe that cannot be accessed in the future, and to reclaim resources, particularly memory, used by those objects. Most garbage-collected language uses either reference counting or tracing .

Python uses reference counting , which has the benefit that it can immediately reclaim objects when their reference count goes to 0.

  • A challenge of reference counting is reference cycles . For example, objects A and B reference each other, A.u=B, B.v=A. The reference count will never below 1 , even if nothing else reference A and B . In Python, garbage collector will periodically check this, and removes them.

  • Garbage collector uses heuristics for speed. Empirically, the recent created objects are more likely to be dead. That is younger “generations” are examined first.

Tracing, which is used in Java, can be performed in seperate thread to make it higher performance, but also has a weakness: when the garbage collector runs, it pauses all thread .

Shallow and Deep Copy

First note that assignment in Python does NOT copy , and it’s simply a variable to a target. For example, if A = [1, 2, 3] , and B is assigned to by A , then changing B will also change A .

The difference between shallow and deep copy is only relevant for compound objects , that is objects that contain other objects. Shallow and deep copy are implemented in copy module. A shallow copy contructs a new compound object and inserts references into it to the objects found in the original. A deep copy contructs a new compound object and recursively insert copies into it of the objects found in the original.

For example, if A = [1, 2, 3] , then shallow and deep copy will make no difference, because it’s not a compound objects. And if A = [[1, 2], [3, 4], [4, 5]] and B = copy.copy(A) , we update B[0][0] will also change A[0][0]. But if B = copy.deepcopy(A), then updating B[0][0] will leave A[0][0] unchanged.

Deep copy is more challenging to implement since recursive objects result in infinite recursion in na√Øve implementations of deep copy. Furthermore, since a deep copy copies everything between copies, it may copies too much. The copy.deepcopy() function avoids these problems by caching objects that already copied, and letting user-defined classes override the copying function. In particular, a class can override one or both of __copy__() and __deepcopy__().

Copying is defensive – it may avoided if it’s known that the client will not mutate the object. Similarly, if the object itself is immutable, then there’s no need copying it.

Closure

What does the following program print and why?

1
2
3

increment_by_i = [lambda x: x + i for i in range(10)]

print(increment_by_i[3](4))

It will print 13(9+4) instead of 7(3 + 4);

After the first line, i became 9, and since increment_by_i is in the form of [lambda x: x+i, ..., lambda x: x+i]. So increment_by_i is actually [lambda x: x+9, ..., lambda x: x+9].

To get the desired behaviour, a reasonable approach is to return the lambda from a function, thereby avoiding the naming conflict.

1
2
3
4

def create_increment_function(x):
    return lambda y: y+x

increment_by_i = [create_increment_function(i) for i in range(10)]

def time_function(f):
    def wrapper(*args, **kwargs):
	before = time.time()
	result = f(*args, **kwargs)
	after = time.time()
	print("Function {} call took ".format(f.__name__) + str(after - before) + " seconds to execute.")
	return result
    return wrapper

The annotation @time_function amounts to the following: foo = @time_function(foo).

Scoping (Explain the rules for the variable scope)

There are two possible (nonexclusive) possibility: the varaible appears in expression, and the variable is being assigned to.

When the varibale appears in expression, python searches for it in the following order.

  1. The current function

  2. Enclosing scoeps, e.g., containing functions

  3. The module that contains the code (also referenced as the global scope)

  4. The built-in scope, e.g. open (???)

A NameError is raised if none of these contains a defined variable with the given name.

To make an assignment to a varibale not defined in the current scope within a function, define a variable in the current scope, that variable should be declared as global in the function.

x, y, z = 'global-x', 'global-y', 'global-z'

def basic_scoping():
    print(x) # expression: global-x
    y = 'local-y' # assignment, y here is defined as a local variable, will not change 'global-y'
    global z
    z = 'local-z' # z declared as 'global', this will change 'global-z'

basic_scoping()
print(x, y, z) # 'global-x', 'global-y', 'local-z'

def inner_outer_scoping():
    def inner1():
	print(x) # outer-x
    def inner2():
	x = 'inner2-x'
	print(x) # inner2-x
    def inner3():
	nonlocal x # wont effect the module (global) value
	x = 'inner3-x'
	print(x) # inner3-x

    x = 'outer-x'
    inner1(), inner2(), inner3()
    print(x) # inner3-x

inner_outer_scoping()
print(x, y, z) # 'global-x', 'global-y', 'local-z'

Function Arguments

What are the positional, keyword, and default argument to a function?

The following function can be called using positional or keyword arguments.

1
2
3
4
5
6

def foo(x, y, z):
    return x*y*z
foo(1, 2, 3) # all three are positional arguments
foo(1, 2, z=3)
foo(1, y=2, 3) # syntax error: keyword arguments must follow positional arguments
foo(1, x=2, y=3, z=4) # syntax error: a variable can't be assigned twice.

When function is defined, its arguments can be specified to have default values. If the function is called with the argument, the argument gets its default value.

One subtlety with default arguments is that they are evaluated once, specifically when the module is loaded, and are shared across all callers.

1
2
3
4
5
6

def foo(x=[]):
    x.append(1)
    return x

print(foo()) # [1]
print(foo()) # [1, 1]

For this reason, as a rule, mutable arguments should have None as their default values.

2
3

def foo(x=None):
    if x is None:
	x = []
    x.append(1)
    return x

print(foo()) # [1]
print(foo()) # [1]
